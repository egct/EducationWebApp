Introducción


La computación gráfica es una de las áreas de la informática visual, donde se utilizan las computadoras para generar imágenes o gráficos sintéticamente para integrar o cambiar la información visual y espacial del mundo real. La computación gráfica se aplica en la creación de: motores gráficos, motores de videojuegos, modelamiento de objetos 2D y 3D, efectos especiales para películas, procesamiento de imágenes, procesamiento de video, visión por computadora, entre otros (Fuente).

El padre de la computación gráfica es el científico de la computación Ivan Sutherland. A Ivan Sutherland le tomó varios años desarrollar el Sketckpad que fue un programa informático creado en 1963 que permitía la manipulación directa de objetos gráficos, así como la interacción persona-computadora y fue el predecesor de los programas de diseño asistido por computadora o computer-aided design (CAD). Ivan Sutherland recibió a lo largo de su vida recibió varios premios y reconocimientos como el Premio Turing de la Asociación de Maquinaria Computacional, el cual es considerado el premio más importante en las ciencias de la computación, en 1988. En el 2012 recibió el Premio Kioto de Tecnología Avanzada por “logros pioneros en el desarrollo de la computación gráfica e interfaces interactivas”. Además, este científico de la computación es miembro de Academia Nacional de Ingeniería de Estados Unidos, así como de la Academia Nacional de Ciencias de Estados Unidos (Fuente).

La computación gráfica cubre conceptos y aspectos relacionados a las matemáticas, cálculo y geometría aplicada a la computación. En muchas aplicaciones se necesitan manejar puntos, líneas, figuras geométricas y polígonos para representar objetos y modelos en 2D y 3D.

Píxeles, Coordenadas y Figuras Geométricas

La forma más conveniente de especificar un segmento de línea en la pantalla de una computadora es proporcionando las coordenadas de sus dos puntos finales. En matemáticas, las coordenadas son números reales, pero las rutinas primitivas de dibujo de líneas pueden requerir que sean enteros. Este es el caso, por ejemplo, en el lenguaje C#, que se utilizará a lo largo de este libro. El componente o subsistema GDI (Graphics Device Interface) trabaja junto con el núcleo y la API de Windows. Esta interfaz de programación de aplicaciones se encarga del control gráfico de los dispositivos de salida, como los monitores o las impresoras. Las tareas más comunes de GDI son:

  • El dibujo de líneas, curvas, polígonos;
  • El relleno de cuadros, círculos, polígonos, etcétera;
  • Se encarga del renderizado de fuentes y textos;
  • Se encarga del manejo de paletas de colores.

Figura 1.4. Dibujo de una elipse.

Para graficar un círculo se utiliza el contexto g con la función DrawEllipse, y se pueden utilizar cinco parámetros como son la pluma de dibujo, las coordenadas iniciales a partir de donde se grafica el círculo. En este caso el valor del ancho y del largo de la elipse son iguales y equivalentes al diámetro del círculo, por lo que se coloca en ambos parámetros el valor del diámetro del círculo, puesto que el círculo es un caso especial de la elipse.



Figura 1.5. Dibujo de un círculo a partir de una elipse.

El eje de coordenadas por pantalla del mundo de la computación gráfica es un eje en el cual se conserva el eje positivo de las ‘x’ hacia la derecha tal y como el del mundo real, pero el eje positivo de las ‘y’ es hacia abajo, debido a que el estándar de fabricación de los monitores y pantallas se basa en el movimiento de los electrones desde arriba hacia abajo y desde la izquierda hacia la derecha aplicando los principios de la energía potencial eléctrica, donde el origen de coordenadas C(0,0) está en la esquina superior izquierda.

Resumen


  • El compilador de Java traduce el código fuente a código bytecode. El intérprete luego combina el código objeto para producir el programa ejecutable que puede correr sobre un sistema operativo.
  • Cada programa en lenguaje Java tiene una función principal (main), la cual define el punto de entrada a un programa.
  • El archivo de la librería de clases System y Scanner contiene un sinnúmero de funciones para entrada y salida de datos. Entre las funciones más importantes para esta sección están la función ‘printf()’, la función ‘println()’, la función ‘print()’, que son funciones de salida de datos; y el objeto de tipo ‘Scanner’ que maneja varias funciones para entrada de datos.
  • Una variable permite almacenar un elemento en la memoria de acceso aleatorio (RAM) un valor de algún tipo de dato definido en el lenguaje Java.
  • Los cinco operadores aritméticos binarios que maneja el lenguaje Java son: suma (+), resta (-), multiplicación (*), división (/), módulo (%).
  • Los seis operadores relacionales que maneja el lenguaje Java son: igual a (==), no igual a (!=), mayor que (>), menor que ( <), mayor o igual que (>=), menor o igual que ( <=).
  • Los tres operadores lógicos que maneja el lenguaje Java son: operador lógico AND (&&), operador lógico OR (||), operador lógico de negación NOT (!).
  • La clase Math, es una librería que tiene funciones para realizar operaciones matemáticas básicas, tales como: funciones trigonométricas, funciones exponenciales, funciones logarítmicas, funciones para calcular la raíz cuadrada, funciones para calcular el valor absoluto de un número, etc.
  • En el lenguaje Java se pueden crear programas que utilicen operadores aritméticos, relacionales y lógicos para resolver problemas de programación.
